# Commit Message Standard

Goal: keep history readable for public users.

## Format

Use a short, imperative summary in sentence case.

```
Type: short summary
```

- Keep the summary <= 72 characters when possible.
- No trailing period.
- Optional scope in parentheses if it adds clarity:

```
Type(scope): short summary
```

## Recommended Types

- Add: new capability or artifact
- Update: change or improve existing behavior/content
- Fix: bug fix or correction
- Docs: documentation-only changes
- Chore: maintenance or cleanup with no behavior change
- Refactor: internal restructuring with no behavior change
- Remove: delete code/assets/content
- Build: build tooling/config changes
- CI: pipeline-related changes
- Test: test-only changes

## Examples

- Add: generate research index
- Update: homepage hero copy
- Fix: broken research link
- Docs: update README
- Chore: cleanup unused assets
- Refactor(research): simplify index generation
