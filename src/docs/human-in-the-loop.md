# Human-in-the-Loop Control

- Humans decide on scope and risk.
- Automation handles mechanical steps.
- Approvals happen at defined checkpoints.
- Every approval should create an audit artifact.
- Failure is experience, not waste. Capture what happened.

## Practical checkpoints

- Risky changes require explicit approval.
- Production execution requires a gate.
- Verification output is attached to the approval.

## Flow (placeholder)

- Draft → Review → Approve → Execute → Report
